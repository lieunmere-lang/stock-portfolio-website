# Portfolio Website Requirements

## 1. 기능 요구사항

1. 포트폴리오 통합 조회
   - 전체 자산 총액(`Total Value`) 표시
   - 전체 손익(`Total Profit/Loss`) 표시
   - 전체 손익률 표시

2. 자산별 상세 정보
   - 자산 이름
   - 티커(symbol)
   - 보유 수량
   - 평균 매수가
   - 현재가
   - 평가 금액
   - 평가 손익
   - 손익률

3. 데이터 동기화
   - 수동 동기화 버튼(`Sync Now`)
   - 자동 주기 동기화(현재 1시간 간격)

4. API 인터페이스
   - `GET /api/portfolio`: 포트폴리오 데이터 반환
   - `POST /api/sync`: 동기화 트리거

5. UI/UX
   - 로딩 상태 표시(스켈레톤)
   - 데이터 로드 실패 시 사용자 안내
   - 손익 플러스/마이너스 색상 구분
   - 카드 + 테이블 기반 대시보드

## 2. 비기능 요구사항

1. 반응형/모바일 친화적 디자인
2. 프론트엔드와 백엔드 분리
3. CORS 허용으로 로컬 개발 환경 지원
4. 환경 변수로 API 키 관리
5. 데이터가 없거나 API 키가 없는 경우에도 안전하게 동작
6. 유지보수 가능한 코드 구조

## 3. 현재 구현 상태에 맞춘 추가 요구사항

1. 현재는 Upbit 자산만 처리됨
   - 키움증권 연동은 추가 구현 필요
2. 데이터 저장소는 아직 인메모리
   - 장기 저장/이력 조회 미구현
3. 시각화 기능(차트, 자산 비중) 추가 가능
4. 더 나은 UX를 위해
   - 요약 카드 아이콘/색상 개선
   - 테이블 정렬/검색 기능
   - 자산 유형(주식/코인) 구분
