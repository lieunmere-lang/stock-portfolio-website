# Discord 금융 봇 설계 스펙

## 개요

키움증권 & 업비트 통합 포트폴리오 웹사이트에 Discord 봇을 추가하여, 일일 뉴스 리포트/시장 지표/가격 알림을 디스코드 서버 채널로 전송한다.

## 핵심 결정사항

- **봇 방식**: discord.py 라이브러리 기반 봇 1개
- **전달 방식**: 디스코드 서버 채널 (DM 아님)
- **아키텍처**: Cog(모듈) 기반 플러그인 구조 — 향후 일정/기타 모듈 확장 가능
- **채널 구성**: 기능별 분리 (#뉴스리포트, #가격알림, #시장지표)
- **알림 설정**: 슬래시 명령어 + 웹 UI 둘 다 지원
- **기존 파이프라인 재활용**: 뉴스 수집/분석은 기존 코드 그대로 사용

## 디렉토리 구조

```
backend/
├── main.py              # FastAPI + Discord Bot 함께 실행
├── scheduler.py         # APScheduler (기존 + 디스코드 전송 추가)
├── database.py          # DB 모델 (기존 + BotConfig 추가)
├── routers/             # REST API (기존)
├── services/            # 비즈니스 로직 (기존)
│
└── bot/                 # 디스코드 봇 (새 디렉토리)
    ├── __init__.py
    ├── client.py        # 봇 클라이언트 설정, Cog 자동 로드
    ├── config.py        # 채널 ID, 봇 설정 관리
    └── cogs/            # 기능 모듈들
        ├── __init__.py
        ├── finance.py   # 금융 모듈 (뉴스, 알림, 지표)
        └── (향후 schedule.py 등 추가)
```

## 전체 아키텍처

```
FastAPI 서버 (기존)
  ├── APScheduler (기존)
  │   ├── 뉴스 수집/분석 (매일 08:50)  ← 기존 코드
  │   ├── 가격 알림 체크 (5분마다)     ← 새로 추가
  │   └── 시장 지표 전송 (매일 09:00)  ← 새로 추가
  │
  ├── Discord Bot (새로 추가)
  │   ├── 서버 시작 시 비동기로 같이 실행
  │   ├── 슬래시 명령어 수신
  │   └── 채널에 메시지 전송
  │
  └── 웹 UI (기존 프론트엔드)
      └── 가격 알림 설정 페이지 ← 새�� 추가
```

- discord.py 봇은 FastAPI와 같은 프로세스에서 비동기로 실행
- 각 Cog은 독립적이며 자기 채널 ID만 알면 됨
- 모듈 추가 = cogs/ 폴더에 파일 추가 → 자동 로드

## 금융 모듈 (finance.py Cog) 상세

### 1. 뉴스 리포트 기능

- **스케줄**: 매일 08:50 기존 스케줄러가 뉴스 수집/분석 → DB 저장. 09:00 봇이 DB에서 리포트 조회 → #뉴스리포트 채널에 임베드 전송
- **메시지 형식**: Discord Embed
  - 제목: `📊 모닝 리포트 — YYYY-MM-DD`
  - 카테고리별 필드: 매크로, 주식, 크립토
  - 각 항목: 제목 + 중요도(⭐)
- **데이터 소스**: 기존 `NewsReport` + `NewsReportItem` 테이블 재활용
- **AI 비용**: Claude Haiku 분석 1회/일 — 약 $0.03~0.05 (40~70원)

### 2. 가격 알림 기능

- **스케줄**: 5분마다 APScheduler가 보유 자산 가격 체크
- **동작**: 조건 충족 시 → #가격알림 채널에 즉시 임베드 전송
- **조건 타입**: "이상" (above) / "이하" (below)
- **알림 설정 방법**:
  - 슬래시 명령어: `/알림추가 BTC 이상 100000000`
  - 웹 UI: 프론트엔드 알림 설정 페이지 (REST API 공유)
- **AI 비용 없음**: 단순 숫자 비교, Claude 불필요

### 3. 시장 지표 기능

- **스케줄**: 뉴스 리포트와 함께 매일 09:00
- **채널**: #시장지표
- **지표 목록**: 공포탐욕지수, VIX, BTC 도미넌스, 원/달러 환율
- **메시지 형식**: Discord Embed로 한눈에 보기 좋게 포맷
- **AI 비용 없음**: 외부 API 조회 후 포맷팅만

## 슬래시 명령어

| 명령어 | 설명 | 예시 |
|--------|------|------|
| `/알림추가 [종목] [조건] [가격]` | 가격 알림 등록 | `/알림추가 BTC 이상 1억` |
| `/알림목록` | 설정된 알림 목록 조회 | `/알림목록` |
| `/알림삭제 [번호]` | 알림 삭제 | `/알림삭제 3` |
| `/리포트` | 최신 뉴스 리포트 즉시 조회 | `/리포트` |
| `/지표` | 현재 시장 지표 즉시 조회 | `/지표` |

## DB 모델

### 기존 모델 수정

**PriceAlert** 테이블에 필드 추가:
- `notification_channel` (String): "discord" / "web" — 알림 전달 대상

### 새 테이블: BotConfig

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | Integer, PK | 자동 증가 |
| `module_name` | String | 모듈 식별자 (예: "finance.news") |
| `channel_id` | String | 디스코드 채널 ID |
| `config_json` | JSON | 모듈별 설정 |
| `is_active` | Boolean | 활성화 여부 |

예시 데이터:
- `module: "finance.news", channel: #뉴스리포트, config: {send_time: "09:00"}`
- `module: "finance.alert", channel: #가격알림, config: {}`
- `module: "finance.indicator", channel: #시장지표, config: {send_time: "09:00"}`

향후 확장 예시:
- `module: "schedule.reminder", channel: #리마인더, config: {}`

## 환경변수

`.env`에 추가:
```
DISCORD_BOT_TOKEN=    # Discord Developer Portal에서 발급
DISCORD_GUILD_ID=     # 디스코드 서버 ID
```

채널 ID는 DB(BotConfig)에서 관리하므로 환경변수는 이 2개만 필요.

## 실행 흐름

```
서버 시작
  ├── FastAPI 실행 (기존)
  └── Discord Bot 실행 (비동기)
       └── finance Cog 로드

매일 08:50 → 뉴스 수집/분석 → DB 저장
매일 09:00 → #뉴스리포트에 임베드 전송
           → #시장지표에 지표 임베드 전송
5분마다    → 가격 체크 → 조건 충족 시 #가격알림에 전송
사용자 명령 → 슬래시 명령어로 알림 설정/조회/삭제
```
